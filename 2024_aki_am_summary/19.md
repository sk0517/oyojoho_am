# 令和6年度秋期 問19（コンピュータシステム）

## 問題文

分散開発環境において，各開発者のローカル環境に全履歴を含んだ中央リポジトリの完全な複製をもつことによって，中央リポジトリにアクセスできないときでも履歴の調査や変更の記録を可能にする，バージョン管理ツールはどれか。

ア　Apache Subversion

イ　CVS

ウ　Git

エ　RCS

## 使用画像

なし

## 解答と解説

**正解：ウ**

Gitは分散型バージョン管理システムであり、各開発者のローカル環境（ローカルリポジトリ）に、コミット履歴を含むリポジトリ全体の完全な複製（クローン）を保持する。そのため、中央（リモート）リポジトリにネットワーク接続できない状況でも、ローカルで履歴の調査、コミット（変更の記録）、ブランチ操作などが可能である。これはGitの分散型アーキテクチャの大きな特徴であり、ウが正解である。

- ア（Apache Subversion）とイ（CVS）は集中型バージョン管理システムであり、履歴は中央のリポジトリのみが保持し、クライアントは作業コピー（ワーキングコピー）のみを持つ。中央リポジトリにアクセスできないと履歴調査やコミット操作ができない。
- エ（RCS：Revision Control System）は単一ファイルを対象としたローカルのバージョン管理ツールであり、分散開発環境や中央リポジトリという概念自体を持たない。

以上より、正解はウである。

**IPA公式：ウ**
