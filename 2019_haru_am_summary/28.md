# 平成31年度春期 問28（技術要素）

## 問題文

過去3年分の記録を保存している“試験結果”表から，2018年度の平均点数が600点以上となったクラスのクラス名と平均点数の一覧を取得するSQL文はどれか。ここで，実線の下線は主キーを表す。

試験結果（<u>学生番号</u>，<u>受験年月日</u>，点数，クラス名）

ア　SELECT クラス名, AVG(点数) FROM 試験結果
GROUP BY クラス名 HAVING AVG(点数) &gt;= 600

イ　SELECT クラス名, AVG(点数) FROM 試験結果
WHERE 受験年月日 BETWEEN '2018-04-01' AND '2019-03-31'
GROUP BY クラス名 HAVING AVG(点数) &gt;= 600

ウ　SELECT クラス名, AVG(点数) FROM 試験結果
WHERE 受験年月日 BETWEEN '2018-04-01' AND '2019-03-31'
GROUP BY クラス名 HAVING 点数 &gt;= 600

エ　SELECT クラス名, AVG(点数) FROM 試験結果
WHERE 点数 &gt;= 600
GROUP BY クラス名
HAVING (MAX(受験年月日)
BETWEEN '2018-04-01' AND '2019-03-31')

## 使用画像

なし

## 解答と解説

**正解：イ**

求めたいのは「2018年度（2018-04-01～2019-03-31）の受験データに限定した上で、クラスごとの平均点数が600点以上となるクラス名と平均点数」である。この条件を満たすには、次の2つが必要になる。

1. WHERE句で受験年月日を2018年度の範囲（'2018-04-01' ～ '2019-03-31'）に絞り込む（過去3年分のデータから対象年度分だけを抽出するため）
2. GROUP BY クラス名でクラスごとに集約し、HAVING AVG(点数) >= 600 で平均点数が600点以上のクラスだけを残す

イはこの両方を満たしており正解。

- ア：WHERE句で年度を絞り込んでいないため、過去3年分全体の平均点数で判定してしまい、「2018年度の平均点数」という条件を満たさない。
- ウ：HAVING句が「点数 >= 600」となっており、集約関数AVGを使っていない。個々の点数に対する条件になり、SQLとしても平均点数の判定にならない（多くのDBMSでは非集約列の単純比較はエラーまたは意図しない結果になる）。
- エ：WHERE句が「点数 >= 600」で個々のレコードを事前に絞り込んでから平均を取っており、600点未満の受験を除外した上での平均になるため、正しい「クラス全体の平均点数」にならない。また年度の絞り込み条件もHAVING句内で不適切に使われている。

よって正解はイ。
