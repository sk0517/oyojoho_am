# 平成31年度春期 問27（技術要素）

## 問題文

RDBMSにおいて，特定の利用者だけに表を更新する権限を与える方法として，適切なものはどれか。

ア　CONNECT文で接続を許可する。

イ　CREATE ASSERTION文で表明して制限する。

ウ　CREATE TABLE文の参照制約で制限する。

エ　GRANT文で許可する。

## 使用画像

なし

## 解答と解説

**正解：エ**

RDBMSにおいて、特定の利用者に対して表への操作権限（SELECT、INSERT、UPDATE、DELETEなど）を付与するには、SQLのアクセス制御文であるGRANT文を使用する。例えば「GRANT UPDATE ON 表名 TO 利用者名」とすることで、指定した利用者だけに更新権限を与えられる。

- ア：CONNECT文はデータベースへの接続を行う文であり、表単位の更新権限の付与ではない。
- イ：CREATE ASSERTION文は、複数の表にまたがる整合性制約（表明）を定義する文であり、権限付与とは無関係。
- ウ：CREATE TABLE文の参照制約（REFERENCES）は外部キー制約を定義するものであり、利用者への権限付与の仕組みではない。

よって、利用者に更新権限を与える適切な方法はエのGRANT文である。

**IPA公式：エ**
