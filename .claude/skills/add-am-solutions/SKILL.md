# 午前問題 解答・解説 追記スキル

## 背景

`C:\WorkSpace\oyojoho_am\` には年度・季節ごとに `{年度}_{季節}_am_summary\` フォルダがあり、各フォルダに問1〜問80のMDファイル（`01.md`〜`80.md`）が入っている。各MDはフロントマター＋「## 問題文」＋選択肢（ア〜エ）＋「## 使用画像」で構成されるが、**正解・解説は含まれていない**。このスキルは、各ファイルの末尾に正解と解説を追記する作業を、正確かつ高速に繰り返すための手順。

進捗は `C:\WorkSpace\oyojoho_am\TODO.md` に記録する。作業を始める前に必ずこのファイルを読み、どの年度・季節が未着手かを確認すること。

## 全体の流れ

1. **対象バッチを確認する** — TODO.mdを見て未着手の年度・季節を洗い出し、複数候補がある場合はユーザーに着手順を確認する（80問×21期という規模のため、一度に全部やろうとしない）
2. **正解キーを用意する** — `answer/answers.json` に該当PDFの正解が既に入っているか確認する。無ければ抽出する
3. **サブエージェントに並行で作業させる** — 80問を4分割（1-20/21-40/41-60/61-80）し、Agentツールを1メッセージ内で4並列起動する
4. **サンプル確認** — 完了後、数ファイルを実際に読んで解説の質・書式をチェックする
5. **TODO.md更新** — 完了した年度・季節にチェックを入れる

## ステップ1: 正解キーの用意

正解データは `answer/{年度コード}_ap_am_ans.pdf`（IPA公式解答）にある。既に `answer/extract_answers.py` があり、実行すると全PDFから正解を機械抽出して `answer/answers.json` に出力する（PDFにテキスト層があるもの全て対応済み）。

```bash
cd "C:/WorkSpace/oyojoho_am/answer" && python extract_answers.py
```

一部のPDF（例: 2022年春）は画像スキャンでテキスト抽出できない。その場合は `answer/answers.json` に空辞書として記録されるので、TODO.mdにも要OCRである旨を明記し、その期はスキップするか、OCR（`pdf2image` + 目視 or OCRツール）で正解を確認してから進める。自己判断で正解を推測しないこと — IPA公式解答が唯一の正解ソース。

正解を確認したら、対象期の80問分の「問題番号: 正解」マッピングを控えておく（サブエージェントへの指示に埋め込むため）。

## ステップ2: サブエージェントの並行起動

**なぜ分割・並列化するか**: 80問を1つのエージェントで順に処理すると時間がかかる上、コンテキストが肥大化して後半の解説品質が落ちやすい。4エージェントに20問ずつ渡すと、速く終わり、各エージェントが自分の担当範囲に集中できる。

1メッセージの中でAgentツールを4回呼び出す（独立したタスクなので並列実行される）。各エージェントへのプロンプトには以下を必ず含める:

- 対象フォルダのパスと対象ファイル範囲（例: `01.md`〜`20.md`）
- 正解マッピング（問題番号:正解記号の一覧をそのまま貼り付ける）
- 追記フォーマット（下記「解答セクションの書式」をそのまま貼り付ける）
- 「正解記号は必ず指定データに従うこと。自分で考えて選ばない」という注意
- 「問題文・選択肢・使用画像は変更せず、末尾に追記のみ」という注意
- 完了したら処理ファイル数を報告するよう指示

過去の実行では、計算問題（待ち行列、EOQ、LP最適化、自己資本比率など）は画像内の数値を読み取って実際に検算させると精度が上がる。用語問題は選択肢の意味を簡潔に説明させる。

### 解答セクションの書式

各ファイル末尾に、既存内容はそのまま残して以下を追記する:

```markdown
## 解答と解説

**正解：{ア/イ/ウ/エ}**

{解説文。問題文中の条件・計算過程・キーワードの意味などを踏まえ、なぜその選択肢が正解かを論理的に説明する。誤答選択肢にも簡潔に触れてよいが、1問あたり10〜20行程度に収める}

**IPA公式：{ア/イ/ウ/エ}**
```

この書式は `C:\WorkSpace\code\PmExam\` の午後問題ファイル（「## 解答と解説」→ 正解の理由 → IPA公式、の順）に合わせたもの。統一感を保つため変更しない。

解説の中身はサブエージェント自身のIT知識で生成する（外部サイトへのアクセスは不要）。正解記号だけは必ずIPA公式解答（answers.json）に従わせる。

## ステップ3: 全件の正解記号を機械検証する（省略しないこと）

**重要な既知の落とし穴**: サブエージェントは、問題文を自分で解いた結果が指示した正解データと異なると判断すると、指示を無視して自分の解釈の答えを書いてしまうことがある（実績: 80問中2問でこれが発生）。サンプル確認だけでは見逃すので、必ず80問全件を機械的に照合する。

対象フォルダで以下のようなスクリプトを実行し、各ファイルの `**正解：X**` を抽出して `answers.json` の値と全件突き合わせる:

```python
import re, json
d = json.load(open("../answer/answers.json", encoding="utf-8"))
expected = d["{該当PDFファイル名}"]
mismatches = []
for n in range(1, 81):
    text = open(f"{n:02d}.md", encoding="utf-8").read()
    m = re.search(r"\*\*正解[：:]\s*([アイウエ])", text)
    got = m.group(1) if m else None
    exp = expected[str(n)]
    if got != exp:
        mismatches.append((n, got, exp))
print(mismatches)
```

- `grep -c "## 解答と解説"` が2以上のファイルがないか（重複追記チェック）も合わせて確認する
- mismatchesが見つかった場合、該当ファイルの問題文・選択肢・（あれば画像）を読み直し、なぜ指定の正解になるのかを正しく解き直してから、Editで解説文と正解記号（本文中とIPA公式の両方）を修正する。画像付き問題は`Read`で画像を直接見て内容を確認すること
- 数ファイルは実際に内容を読んで、解説の質（具体性、計算過程の有無）も確認する

## ステップ4: TODO.md更新

`TODO.md` の対象年度・季節の行を「✅ 完了（80/80、{日付}）」のように更新する。次に着手すべき年度・季節をユーザーに確認する一言を添える。

## 注意点

- 1回のスキル実行で処理するのは基本1期（80問）まで。複数期をまとめて頼まれた場合も、期ごとに上記フローを繰り返す（コンテキストとエージェント数の管理のため）
- OCRが必要な期（現状: 2022年春）は先に正解確認の方法をユーザーと相談してから着手する
- 2019_aki フォルダの有無など、フォルダ構成に疑問があれば作業前に`ls`で確認する
