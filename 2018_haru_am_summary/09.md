# 平成30年度春期 問9（コンピュータシステム）

## 問題文

プロセッサの高速化技法の一つとして，同時に実行可能な複数の動作を，コンパイルの段階でまとめて一つの複合命令とし，高速化を図る方式はどれか。

ア　CISC

イ　MIMD

ウ　RISC

エ　VLIW

## 使用画像

なし

## 解答と解説

**正解：エ**

VLIW（Very Long Instruction Word）は、コンパイラが同時に実行可能な複数の命令（動作）を解析し、コンパイル時にそれらをあらかじめ一つの長い複合命令語にまとめておく方式である。実行時には、プロセッサはこの複合命令を並列に実行することで、実行時の命令スケジューリングにかかるオーバーヘッドを削減し高速化を図る。これは選択肢エの内容と一致する。

他の選択肢は次のとおり異なるプロセッサアーキテクチャを指す。
- ア：CISC（Complex Instruction Set Computer）は、複雑で多機能な命令を数多く持つアーキテクチャである。
- イ：MIMD（Multiple Instruction stream, Multiple Data stream）は、複数のプロセッサがそれぞれ異なる命令を異なるデータに対して実行する並列処理方式の分類である。
- ウ：RISC（Reduced Instruction Set Computer）は、命令セットを単純化・縮小し、1命令1サイクルでの実行を目指すアーキテクチャである。

したがって、コンパイル段階で複数の動作をまとめて一つの複合命令とする高速化技法はVLIWであり、選択肢エが正解である。

**IPA公式：エ**
