# 令和3年度春期 問29（技術要素）

## 問題文

商品の注文を記録するクラス（顧客，商品，注文，注文明細）の構造を概念データモデルで表現する。a〜dに入れるべきクラス名の組合せはどれか。ここで，顧客は何度も注文を行い，一度に一つ以上の商品を注文でき，注文明細はそれぞれ1種類の商品に対応している。また，モデルの表記にはUMLを用いる。

## 解答と解説

**正解：ア**

図はUMLのクラス図で、a(1)—(*)b(1)—(*)c(*)—(1)d という多重度をもつ関連が示されている。問題文の条件を整理すると次のようになる。

- 顧客は何度も注文を行う → 顧客(1) と 注文(*) の関係（顧客１人に対し注文は複数）
- 一度の注文で一つ以上の商品を注文できる → 注文(1) と 注文明細(*) の関係（注文１件に対し注文明細は複数）
- 注文明細はそれぞれ1種類の商品に対応 → 注文明細(*) と 商品(1) の関係（複数の注文明細が同一商品に対応）

これを図の並び a(1)-(*)b(1)-(*)c(*)-(1)d に当てはめると、a＝顧客、b＝注文、c＝注文明細、d＝商品となる。

- イ：a＝商品としているが、a側は「1」で始まり顧客側の多重度と整合しない。
- ウ：b＝注文明細、c＝顧客としており、顧客が複数存在し得る（*側）位置と矛盾する。
- エ：a＝注文明細、d＝顧客としており、顧客が(1)側に置かれるべき位置と矛盾する。

したがって、a＝顧客、b＝注文、c＝注文明細、d＝商品の組合せであるアが適切である。

**IPA公式：ア**
