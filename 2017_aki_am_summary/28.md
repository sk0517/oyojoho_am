# 平成29年度秋期 問28（技術要素）

## 問題文

関係R（ID，A，B，C）のA，Cへの射影の結果とSQL文で求めた結果が同じになるように，aに入れるべき字句はどれか。ここで，関係Rを表Tで実現し，表Tに各行を格納したものを次に示す。

〔SQL文〕

　SELECT     a     A, C FROM T

ア　ALL

イ　DISTINCT

ウ　ORDER BY

エ　REFERENCES

## 使用画像

![AP2017AA028-01.gif](images/AP2017AA028-01.gif)

## 解答と解説

**正解：イ**

表Tの内容は次のとおりである。

| ID | A | B | C |
|----|----|----|----|
| 001 | a1 | b1 | c1 |
| 002 | a1 | b1 | c2 |
| 003 | a1 | b2 | c1 |
| 004 | a2 | b1 | c2 |
| 005 | a2 | b2 | c2 |

関係RのA，Cへの射影（重複行を除いた集合演算としての射影）を行うと、(A, C)の組み合わせは
(a1, c1)：001, 003行から重複
(a1, c2)：002行
(a2, c2)：004, 005行から重複
の3行に集約される。

SQLでは、単純に `SELECT A, C FROM T` とすると重複行がそのまま出力されてしまう（5行）。関係代数の射影のように重複を除去した結果を得るには、`SELECT DISTINCT A, C FROM T` のように**DISTINCT**キーワードを指定する必要がある。DISTINCTは検索結果から重複するタプル（行）を取り除くためのSQLキーワードである。

- ア：ALLは重複を許容してすべての行を出力するキーワードであり、射影の重複除去とは逆の効果になる。
- ウ：ORDER BYは結果の並び替えを指定するもので、重複除去とは無関係。
- エ：REFERENCESは外部キー制約を定義する際に使うキーワードであり、SELECT文の重複除去とは無関係。

以上より、重複除去を行うDISTINCTを入れるイが正解である。

**IPA公式：イ**
