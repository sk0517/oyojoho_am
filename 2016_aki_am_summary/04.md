# 平成28年度秋期 問4（基礎理論）

## 問題文

表は，入力記号の集合が｛0，1｝，状態集合が｛a，b，c，d｝である有限オートマトンの状態遷移表である。長さ3以上の任意のビット列を左（上位ビット）から順に読み込んで最後が110で終わっているものを受理するには，どの状態を受理状態とすればよいか。

ア　a

イ　b

ウ　c

エ　d

## 使用画像

![AP2016AA004-01.gif](images/AP2016AA004-01.gif)

## 解答と解説

**正解：ウ**

状態遷移表は次の内容である（開始状態はa）。

| 状態 | 0 | 1 |
|---|---|---|
| a | a | b |
| b | c | d |
| c | a | b |
| d | c | d |

各状態が「これまでに読んだビット列の末尾2桁が何であるか」を表していると仮定して、a＝末尾"00"、b＝末尾"01"、c＝末尾"10"、d＝末尾"11"と対応付けると、すべての遷移がこの解釈と矛盾なく一致する（例：末尾"01"の状態bで0を読めば末尾は"10"になりcへ遷移し、表とも一致する）。

したがって、状態cは「直近に読んだ2ビットが"10"である」ことを表す状態である。ビット列が"…110"で終わっている場合、最後の1を読み込んだ直後の状態は「直近2ビットが10」＝状態cとなる（"…11"を読んだ状態dから最後の"0"を読むとc、あるいは"…10"の状態cから"1"の位置で見ると、最終的に列全体を読み終えた時点の状態がcであれば末尾が"10"、その直前に"1"があれば全体として"110"で終わることが保証される）。

実際に遷移を追うと、状態cへ到達するのは直前の入力が0であり、かつその前の状態がb（末尾01）またはc（末尾10）の場合であり、これは直近2文字が"10"であることを意味する。ビット列の末尾が"110"であるとは、読み終えた時点の状態が「末尾10」（＝直近2ビットが10）であることと同値であるため、受理状態は状態cとすればよい。

**IPA公式：ウ**
